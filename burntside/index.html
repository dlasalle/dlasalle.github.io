Building

Building Burntside requires a C99 compiler supporting OpenMP specification 3.0 or greater (almost all of my testing is with GCC 4.7 and 4.8) and CMake. The configure script requires Bash, however it is possible to do the configuration by hand. To build Burntside with the default options, execute:

./configure && make

After it is built, the binary and the library will reside in ./build/[system-arch]/bin and ./build/[system-arch]/lib respectively (for most systems, [system-arch] will be Linux-x86_64, but depends on your OS/architecture). Burntside can the optionally be installed using make install (the default prefix is /usr/local, but can be changed with configure --prefix="my_prefix").

Usage

Burntside can then be used to cluster a graph as follows:

burntside test.graph

Most users will actually want the generated and not just runtime statistics, and should append the name of the output file.

burntside test.stp test.sol

The Burntside binary currently only supports the Steinlib File Fromat for input (.stp).

Burntside currently has a large number of options, many of which are experimental. For a complete list, use the -h option:

burntside -h
